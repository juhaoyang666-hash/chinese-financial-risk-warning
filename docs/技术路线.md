# 中文金融负面主体风险识别技术路线

作者：10225001426 杨巨豪

## 1. 项目目标

本项目面向金融风控中的舆情早筛场景，构建一个中文金融负面主体识别系统。输入为金融新闻、公告或社交媒体文本，系统需要判断文本中是否存在针对金融主体的负面风险信息，并在存在风险时抽取对应主体名称。

相比普通金融情感分类，本任务更强调实体级风险归因。例如文本中可能同时出现平台、银行、监管机构和个人，模型不仅要判断文本整体是否负面，还要识别负面信息具体指向哪个金融主体。这更接近风控算法中的黑名单线索发现、平台风险预警和人工审核分流。

## 2. 数据路线

主数据集使用 FinCUGE-Instruction，过滤其中 FINNSP 任务样本。该任务对应“金融负面消息及其主体判定”。

当前已完成数据准备：

- 训练集：4000 条
- 验证集：500 条
- 训练集负面主体存在样本：2221 条
- 训练集无负面主体样本：1779 条
- 验证集负面主体存在样本：259 条
- 验证集无负面主体样本：241 条

数据处理脚本为 `scripts/prepare_data.py`，输出统一 JSONL 格式：

```json
{
  "id": "train-0",
  "text": "金融新闻或社交媒体文本",
  "has_negative": true,
  "label": 1,
  "entities": ["小资钱包", "资易贷"],
  "target_json": "{\"has_negative\": true, \"entities\": [\"小资钱包\", \"资易贷\"]}"
}
```

无负面主体样本统一为：

```json
{
  "has_negative": false,
  "entities": []
}
```

该统一 schema 同时服务于传统分类模型、编码器模型和生成式大语言模型，避免不同实验之间标签口径不一致。

## 3. 标签归一化路线

原始 FINNSP 输出并非严格结构化，常见形式包括：

- `有`
- `该文本中不包含负面金融实体。`
- `文中包含的负面主体：小资钱包`
- `负面金融主体包含以下几个：小资钱包;资易贷`

项目在 `src/risk_nlp/schema.py` 中实现归一化：

- 使用规则识别“不包含、没有、未发现、无负面”等无风险表达。
- 使用冒号、分号、顿号、逗号等中文文本常见分隔符抽取主体。
- 输出统一为 `has_negative` 和 `entities`。
- 对 LLM 非法 JSON 输出提供兜底解析，并记录 `invalid_json_rate`。

这一步是后续实验可靠性的基础，也是论文中“数据构成与标注处理”章节的重要内容。

## 4. 模型路线

### 4.1 传统机器学习基线

第一组基线使用字符级 TF-IDF 表示文本，再训练线性分类器：

- TF-IDF + Logistic Regression
- TF-IDF + Linear SVM

该路线训练快、可解释性强，是论文中展示深度模型收益的必要参照。当前已完成两组实验，验证集结果如下：

| 模型 | Accuracy | Precision | Recall | F1 | Macro-F1 |
|---|---:|---:|---:|---:|---:|
| TF-IDF + Logistic Regression | 0.9300 | 0.9242 | 0.9421 | 0.9331 | 0.9299 |
| TF-IDF + Linear SVM | 0.9500 | 0.9432 | 0.9614 | 0.9522 | 0.9499 |

### 4.2 中文预训练编码器基线

第二组模型使用中文预训练编码器完成二分类：

- Chinese RoBERTa-wwm-ext
- Chinese MacBERT-base
- FinBERT2-large

训练脚本为 `scripts/train_encoder.py`。编码器模型相比 TF-IDF 能建模上下文语义，适合作为线上高吞吐风控初筛模型。其输出仍统一转为：

```json
{"has_negative": true, "entities": []}
```

编码器实验主要评估二分类能力；主体抽取能力由生成式 LLM 承担。

当前三组编码器完整训练均已完成，验证集结果如下：

| 模型 | Accuracy | Precision | Recall | F1 | Macro-F1 |
|---|---:|---:|---:|---:|---:|
| Chinese RoBERTa WWM Ext | 0.9540 | 0.9538 | 0.9575 | 0.9557 | 0.9539 |
| Chinese MacBERT Base | 0.9580 | 0.9542 | 0.9653 | 0.9597 | 0.9579 |
| FinBERT2 Large | 0.9620 | 0.9511 | 0.9768 | 0.9638 | 0.9619 |

其中 FinBERT2-large 目前是编码器组最优模型，说明金融领域继续预训练对该任务有明确收益。

### 4.3 大语言模型零样本探索

第三组实验使用 `Qwen3-8B`，不进行参数更新：

- zero-shot：直接给任务说明和 JSON 输出要求。

该实验用于回答：通用指令大模型在没有领域微调时，能否理解金融负面主体识别任务。评估指标除二分类和实体 F1 外，还记录非法 JSON 率和平均推理耗时。

### 4.4 Qwen3-8B QLoRA 主模型

主模型使用 `Qwen3-8B` 进行 QLoRA 指令微调：

- 4-bit NF4 量化
- LoRA rank `r=16`
- LoRA alpha `64`
- LoRA dropout `0.05`
- 目标模块：`q_proj,k_proj,v_proj,o_proj,gate_proj,up_proj,down_proj`
- 最大序列长度：2048
- 训练轮数：3 epoch
- 训练方式：4 卡 DDP，保持等效 batch size

训练脚本为 `scripts/train_llm_qlora.py`。模型输入由系统提示词、用户任务说明和文本组成，训练目标为严格 JSON：

```json
{"has_negative": true, "entities": ["主体A", "主体B"]}
```

QLoRA 的价值在于用较低显存成本完成大模型领域适配。服务器有 8 张 RTX 3090，每张 24GB，适合采用 `accelerate` 进行多卡训练。

## 5. 实验路线

实验按从轻到重的顺序执行：

1. 数据准备与标签归一化。
2. TF-IDF + Logistic Regression / SVM。
3. 中文 RoBERTa、MacBERT、FinBERT2 编码器微调。
4. Qwen3-8B zero-shot 评估。
5. Qwen3-8B QLoRA 微调。
6. LoRA adapter 加载并在验证集上评估。
7. 汇总所有 `metrics.json`，生成论文结果表。
8. 生成标签分布图、文本长度分布图、混淆矩阵。

## 6. 正式训练运行方式

正式训练统一放入 `tmux` 会话运行，避免 SSH 断连影响长时间实验。当前启动的会话如下：

| 会话名 | GPU | 实验 | 日志 |
|---|---:|---|---|
| `nlp_qwen3_zero` | 5 | Qwen3-8B zero-shot 评估 | `logs/qwen3_zero.log` |
| `nlp_qwen3_qlora_main_4gpu` | 0,2,3,4 | Qwen3-8B QLoRA 3 epoch 主方案训练 | `logs/qwen3_qlora_main_train.log` |

常用查看命令：

```bash
tmux ls
tmux attach -t nlp_qwen3_qlora_main_4gpu
tail -f logs/qwen3_qlora_main_train.log
nvidia-smi
```

QLoRA 训练完成后，正式主方案加载低秩适配器进行完整验证集评估，并将结果纳入最终对比表。最终配置参数为 `r=16`、`alpha=64`、`dropout=0.05`、`learning_rate=2e-4`。

## 10. 风控落地解释

最终系统可以拆成两层：

第一层是编码器模型，用于线上大规模舆情流初筛。它速度快、成本低，可以将海量文本筛成少量高风险候选。

第二层是 Qwen3-8B QLoRA 模型，用于高风险候选复核。它能输出结构化 JSON 和负面主体列表，便于接入人工审核、风险工单或企业风险画像系统。

这种“轻量模型初筛 + LLM 精筛”的路线兼顾效率和准确性，也更符合真实风控系统的成本约束。

## 11. 创新性增强路线

为避免论文只停留在“模型对比 + QLoRA 指标提升”，后续建议把项目重点从单模型效果扩展到风控系统可靠性：

- 实体候选约束：要求 LLM 输出的负面主体必须来自文本或候选实体列表，并统计实体在文率、幻觉实体率。
- 风险事件扩展：在主体识别基础上增加风险类型和严重程度，例如诈骗/非法集资、兑付逾期、监管处罚、经营异常等。
- 级联风控流程：用 FinBERT2 或 MacBERT 做全量初筛，将高风险或低置信度样本交给 Qwen3-8B QLoRA 做主体级复核。
- 反事实鲁棒性：构造实体替换、负面词删除、无关主体插入等样本，验证模型是否真正完成主体归因。
- 外部泛化测试：使用 FLARE-zh-NSP 做二分类泛化评估，说明模型在同类金融负面主体数据上的迁移能力。

详细修改建议见 `docs/项目创新性改进建议.md`。
