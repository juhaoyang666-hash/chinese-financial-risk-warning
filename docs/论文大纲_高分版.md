# 基于大语言模型的中文金融负面主体风险识别研究

## 摘要
说明金融舆情和风控早筛场景中，实体级负面风险识别的重要性。介绍本文使用 FinCUGE FINNSP 数据集，对比传统机器学习、中文预训练编码器、金融领域编码器和 Qwen3-8B QLoRA 指令微调模型，并从二分类与实体抽取两个角度评估。

## 1 引言
- 背景：P2P、信托、消费金融、贷款平台等场景中，风险事件常先出现在新闻和社交媒体文本。
- 问题：普通情感分类只能判断文本情绪，不能回答“哪个主体有风险”。
- 贡献：
  - 构建统一的中文金融负面主体识别实验框架。
  - 比较传统方法、编码器模型和 2020 年后大语言模型。
  - 提出面向风控早筛的“编码器初筛 + LLM 主体级复核”级联流程。
  - 从实体幻觉、错误类型、反事实扰动和外部泛化角度分析模型可靠性。

## 2 相关工作
- 通用预训练语言模型：BERT、RoBERTa 奠定了编码器分类基线，MacBERT 代表中文预训练模型改进方向。
- 金融领域预训练与基准：FinBERT 说明领域预训练对金融情感分析有效，BBT-Fin / FinCUGE 提供中文金融 NLP 任务与评测背景。
- 金融大模型与开源生态：BloombergGPT、FinGPT、PIXIU / FinBen 说明金融 LLM 从单一分类走向指令理解、多任务评测和系统化应用。
- 实体级风险识别：强调普通情感分类只能判断整体倾向，负面主体识别需要解决“谁有风险”的归因问题，更适合风控预警。
- 参数高效微调：LoRA、QLoRA 通过低秩适配和 4-bit 量化降低微调成本，是 8 张 3090 上训练 Qwen3-8B 的工程基础。

## 3 数据集与任务定义
- 数据来源：FinCUGE-Instruction 中的 FINNSP 任务。
- 数据规模：训练集 4000 条，验证集 500 条。
- 标签格式：`has_negative` 与 `entities`。
- 任务定义：
  - 二分类：文本是否含有负面金融主体。
  - 主体抽取：抽取负面主体名称。
- 应放入图表：
  - 标签分布图。
  - 文本长度分布图。
  - 数据样例表。

## 4 方法
- TF-IDF + Logistic Regression / Linear SVM。
- 中文编码器：Chinese RoBERTa、MacBERT、FinBERT2。
- Qwen3-8B zero-shot / few-shot。
- Qwen3-8B QLoRA：
  - 4-bit NF4 量化。
  - 主方案训练配置：LoRA r=16, alpha=64, dropout=0.05, learning rate=2e-4。
  - 4 卡 DDP 训练，保持等效 batch size。
  - 输出统一 JSON schema。
- 可作为后续增强的风控模块：
  - 实体候选约束：限制输出主体来自文本或候选实体，降低幻觉主体。
  - 风险事件扩展：增加风险类型、严重程度、证据片段。
  - 级联推理：编码器处理全量文本，LLM 只复核高风险或低置信度样本。

## 5 实验设置与结果分析

### 5.1 实验设置
- 硬件：8 张 NVIDIA RTX 3090, 每张 24GB。
- 软件：PyTorch, Transformers, Datasets, PEFT, BitsAndBytes, TRL。
- 指标：
  - Accuracy, Precision, Recall, F1, Macro-F1。
  - Entity Precision, Entity Recall, Entity F1。
  - LLM invalid JSON rate 与平均推理耗时。
- 建议新增可靠性指标：
  - Entity-in-text rate：预测主体出现在原文中的比例。
  - Hallucinated entity rate：预测主体不在原文中的比例。
  - Cascade LLM call rate：级联系统中需要调用 LLM 的样本比例。

### 5.3 可靠性与错误分析
- 错误类型：
  - 负面词存在但不是针对金融主体。
  - 多主体文本中归因错误。
  - 社交媒体噪声、乱码、URL、话题标签干扰。
  - 长文本中关键信息被截断。
  - LLM 输出了原文中不存在的幻觉主体。
- 放入 5-8 条典型案例，展示模型预测、真实标签和人工分析。
- 当前 Qwen3-8B QLoRA 错误分析结果：
  - False Positive：9 条。
  - False Negative：5 条。
  - Entity Mismatch：72 条。
  - 预测实体在文率：0.9864。
  - 幻觉实体率：0.0136。
- 建议额外加入反事实案例：
  - 替换主体名称，检查模型是否同步改变预测主体。
  - 删除负面触发词，检查模型是否从“有风险”变为“无风险”。
  - 插入无关金融主体，检查模型是否误报。

### 5.4 编码器到 LLM 的级联风控模拟
- 在线风控早筛：编码器模型吞吐高，适合全量文本初筛。
- 离线复核：LLM 适合高风险样本复核、生成解释、辅助人工审核。
- 风险：LLM 可能输出不稳定格式或幻觉主体，需要 JSON 约束、规则校验与人工抽检。
- 成本权衡：报告不同阈值下 LLM 调用比例、召回率和实体 F1，说明两阶段系统比单独部署 LLM 更现实。
- 业务风险：将 False Negative、False Positive、Entity Mismatch 分别对应到漏报、人工审核成本和风险归因误伤。

级联风控模拟结果：

| 设置 | Accuracy | Recall | F1 | Entity-F1 | LLM 调用率 |
|---|---:|---:|---:|---:|---:|
| FinBERT2 only | 0.9620 | 0.9768 | 0.9638 | 0.0000 | 0.0000 |
| Qwen3-8B QLoRA all | 0.9720 | 0.9807 | 0.9732 | 0.8331 | 1.0000 |
| FinBERT2 positive -> Qwen3-8B QLoRA review | 0.9700 | 0.9691 | 0.9710 | 0.8386 | 0.5320 |

可写结论：离线级联模拟中，策略只调用约 53.2% 的 LLM，整体 F1 与全量 LLM 接近，Entity-F1 从 0.8331 提升到 0.8386，说明编码器初筛有潜力减少负样本上的实体误报，更符合高吞吐风控场景。该结果基于已保存预测文件组合，不直接等同线上延迟收益。

## 6 风控系统扩展与应用讨论

### 6.1 系统原型、外部扩展与上线讨论

- 错误案例自动导出：生成 false positive、false negative、entity mismatch 和 multi-entity case 表。
- 实体可信性评估：新增实体在文率、幻觉实体率、多主体错误率。
- 风险事件结构化：将输出从 `entities` 扩展为 `risk_events`。
- 外部泛化：使用 FLARE-zh-NSP 同时报告关键词弱监督 baseline 和 Qwen3-8B QLoRA 真实二分类推理；由于缺少实体 gold，不报告外部 Entity-F1。
- 反事实样本构造：报告规则 baseline 初测和 Qwen3-8B QLoRA 真实推理压力测试；强调自动构造样本不是人工标注鲁棒性基准。

详细方案见：`docs/项目创新性改进建议.md`。
